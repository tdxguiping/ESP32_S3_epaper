串口协议边界

不改现有 WiFi HTTP server。浏览器这个页面通过串口发送 GET /ping HTTP/1.1、POST /delete HTTP/1.1、POST /cast HTTP/1.1 等 HTTP-like 文本。设备端建议新增独立目录 main/usb_console_echo/ 做 transport、http_text、router 和各 feature module。

接收规则：先找 \r\n\r\n，解析 header，按 Content-Length 继续读取 body。不要用“超时即结束”判断请求结束。设备响应统一包在 @#$ 和 %^& 之间，便于 PC 端从调试日志中识别协议响应。

串口连接

USB Serial/JTAG 下 baud 是 PC line coding 参数，不等于 USB 物理吞吐；长期稳定建议 921600 或 1000000，2000000 仅建议压测。

故障保护：串口打开、写入、Reset 或接收过程中如果发生错误/超时，会立即释放 reader/writer、关闭当前串口连接，并继续保持自动重连。
