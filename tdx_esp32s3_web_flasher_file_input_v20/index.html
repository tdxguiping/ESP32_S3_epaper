TDX ESP32-C5 Web Flasher v20

固定 4 个烧写地址，文件名可变。支持自动加载上一级目录固件、Reset Device、手动打开串口控制台；主页面启动后不会自动连接串口。

ESP32-C5 / Web Serial
打开方式

推荐把 4 个 bin 放在本网页目录的上一级目录，然后从上一级目录启动 HTTP 服务。

parent_dir/
├── bootloader.bin
├── partition-table.bin
├── ota_data_initial.bin
├── file_server.bin
└── tdx_esp32s3_web_flasher_file_input_v20/
    └── index.html

然后运行 python -m http.server 8000，用 Chrome / Edge 打开 http://localhost:8000/tdx_esp32s3_web_flasher_file_input_v20/index.html。

如果直接在网页目录里启动服务，浏览器无法访问真实的上一级目录；此时请手动选择 4 个文件。

串口协议测试页面

点击下面按钮会新开 serial_protocol.html，用于通过 USB Serial/JTAG 发送 HTTP-like 串口协议，并包含文件输入窗口，可测试 /upload、/cast、/cast2pic 等发文件功能。

打开串口协议页面前，主页面会自动停止后台串口检测并断开当前串口，避免两个页面抢同一个 COM 口。

备用直接打开 serial_protocol.html

选择烧写文件

第一次打开页面时，会尝试从 index.html 所在目录的上一级目录自动读取：bootloader.bin、partition-table.bin、ota_data_initial.bin、file_server.bin。手动选择文件后，手动文件优先。

Flash 地址 文件用途 手动选择文件 状态
0x2000 Bootloader
自动文件：../bootloader.bin		 未选择
0x8000 Partition Table
自动文件：../partition-table.bin		 未选择
0xD000 OTA Data Initial
自动文件：../ota_data_initial.bin		 未选择
0x20000 Application
自动文件：../file_server.bin，也可手动改成其他 App bin		 未选择

烧写操作

烧写完成后会执行 hard_reset，并自动尝试打开串口控制台接收日志。

Settings

Serial connection configurations:

more information

这些设置参考 ESP Launchpad 的 Settings 页面。烧写波特率用于进入 bootloader 和写 Flash；Console baudrate 用于应用启动后的串口日志。

USB Serial/JTAG 下 baud 是 PC line coding 参数，不等于 USB 物理吞吐；长期稳定建议 921600 或 1000000，2000000 仅建议压测。

自动重连状态：空闲
